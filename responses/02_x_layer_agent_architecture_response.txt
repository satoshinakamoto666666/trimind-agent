### 1. Agent Design Pattern (Answering Q1)
**Hybrid Periodic + Event-Driven** (best for autonomous + responsive).  
- **Periodic loop** (every 30 seconds): Core intelligence (market/signal/trenches/security scans → decision → possible execution). This alone drives massive legitimate OnchainOS volume.  
- **Event-driven**: Discord commands trigger immediate actions (user override, status, withdraw).  
- **Why not pure event-driven?** No reliable on-chain event WS for X Layer in 10 days (your Helius is Solana-only). Periodic + polling skills is perfect and judge-impressive.  
- **Framework**: **LangGraph** (stateful graphs) + LangChain tools. Superior to raw asyncio or AutoGPT for reliability, error recovery, and AI judge scoring (clean, production-grade agentic code). Claude Opus 4.6 as primary LLM for decisions (best reasoning); Grok API as fallback for creative signals.  
- **Multi-agent?** No — single supervisor + executor pattern (simple, fast to build).

### OnchainOS CLI Commands (in wrappers)
All via `subprocess.run(["onchainos", "<skill>", "<action>", "--json", ...])`  
Example (skills/base.py):
```python
def run_skill(skill: str, action: str, **kwargs) -> dict:
    cmd = ["onchainos", skill, action, "--json"] + [f"--{k}={v}" for k,v in kwargs.items()]
    result = subprocess.run(cmd, capture_output=True, text=True)
    return json.loads(result.stdout)
```
